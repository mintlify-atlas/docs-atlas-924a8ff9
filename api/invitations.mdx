## Overview

The Invitations API allows you to invite users to join organizations via email. Invitations are sent through WorkOS and can be managed (resent, revoked, or deleted) by authorized users.

All invitation operations require authentication and appropriate organization permissions.

## Invitation lifecycle

Invitations have the following states:

- `pending` - Invitation sent, awaiting acceptance
- `accepted` - User has accepted the invitation
- `expired` - Invitation has expired (7 days after creation)
- `revoked` - Invitation has been manually revoked

## Create invitations

Creates one or more invitations to an organization. The inviter must have permission to invite users to the organization. This operation sends invitation emails via WorkOS.

<ParamField body="organizationId" type="string" required>
  Organization ID to send invitations for.
</ParamField>

<ParamField body="invites" type="array" required>
  Array of invitation objects.
  
  <Expandable title="properties">
    <ParamField body="invites[].email" type="string" required>
      Email address to send the invitation to.
    </ParamField>
    
    <ParamField body="invites[].role" type="string" required>
      Role to assign: `"member"` or `"admin"`.
    </ParamField>
  </Expandable>
</ParamField>

### Response

<ResponseField name="results" type="array" required>
  Array of invitation results, one per invite.
  
  <Expandable title="properties">
    <ResponseField name="email" type="string" required>
      Email address from the request.
    </ResponseField>
    
    <ResponseField name="success" type="boolean" required>
      Whether the invitation was created successfully.
    </ResponseField>
    
    <ResponseField name="data" type="object">
      Invitation data if successful.
      
      <Expandable title="properties">
        <ResponseField name="id" type="string">
          Invitation ID.
        </ResponseField>
        
        <ResponseField name="invitationUrl" type="string">
          URL to accept the invitation.
        </ResponseField>
        
        <ResponseField name="workosInvitationId" type="string">
          WorkOS invitation ID.
        </ResponseField>
        
        <ResponseField name="organizationId" type="string">
          Organization ID.
        </ResponseField>
        
        <ResponseField name="email" type="string">
          Invitee email address.
        </ResponseField>
        
        <ResponseField name="invitedBy" type="string">
          User ID of the inviter.
        </ResponseField>
        
        <ResponseField name="invitedAt" type="string">
          ISO 8601 timestamp of invitation creation.
        </ResponseField>
        
        <ResponseField name="expiresAt" type="string">
          ISO 8601 timestamp of expiration (7 days from creation).
        </ResponseField>
        
        <ResponseField name="status" type="string">
          Invitation status: `"pending"`, `"accepted"`, `"expired"`, or `"revoked"`.
        </ResponseField>
        
        <ResponseField name="acceptedAt" type="string">
          ISO 8601 timestamp of acceptance (null if not accepted).
        </ResponseField>
        
        <ResponseField name="acceptedBy" type="string">
          User ID who accepted (null if not accepted).
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="error" type="string">
      Error message if the invitation failed.
    </ResponseField>
    
    <ResponseField name="transactionId" type="string">
      Transaction ID if successful.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="successCount" type="number" required>
  Number of successfully created invitations.
</ResponseField>

<ResponseField name="errorCount" type="number" required>
  Number of failed invitations.
</ResponseField>

### Errors

- `UnauthorizedError` - User lacks permission
- `InternalServerError` - Unexpected server error

<Note>
  This endpoint returns batch results. Individual invitation failures do not cause the entire request to fail. Check the `success` field on each result.
</Note>

<CodeGroup>

```typescript TypeScript
const result = yield* client.call("invitation.create", {
  organizationId: "org_123",
  invites: [
    { email: "alice@example.com", role: "admin" },
    { email: "bob@example.com", role: "member" }
  ]
});

console.log(`${result.successCount} invitations sent`);
console.log(`${result.errorCount} invitations failed`);

for (const result of result.results) {
  if (result.success) {
    console.log(`✓ Sent to ${result.email}`);
  } else {
    console.log(`✗ Failed for ${result.email}: ${result.error}`);
  }
}
```

```javascript JavaScript
const result = await client.call("invitation.create", {
  organizationId: "org_123",
  invites: [
    { email: "alice@example.com", role: "admin" },
    { email: "bob@example.com", role: "member" }
  ]
});

console.log(`${result.successCount} invitations sent`);
console.log(`${result.errorCount} invitations failed`);

for (const result of result.results) {
  if (result.success) {
    console.log(`✓ Sent to ${result.email}`);
  } else {
    console.log(`✗ Failed for ${result.email}: ${result.error}`);
  }
}
```

</CodeGroup>

## Resend invitation

Resends an existing invitation via WorkOS. Only the invitation creator or organization admins can resend invitations.

<ParamField body="invitationId" type="string" required>
  Invitation ID to resend.
</ParamField>

### Response

<ResponseField name="data" type="object" required>
  The invitation object (same structure as in create invitations).
</ResponseField>

<ResponseField name="transactionId" type="string" required>
  Transaction ID for optimistic UI updates.
</ResponseField>

### Errors

- `InvitationNotFoundError` - Invitation does not exist
- `UnauthorizedError` - User lacks permission
- `InternalServerError` - Unexpected server error

<CodeGroup>

```typescript TypeScript
const result = yield* client.call("invitation.resend", {
  invitationId: "inv_123"
});
```

```javascript JavaScript
const result = await client.call("invitation.resend", {
  invitationId: "inv_123"
});
```

</CodeGroup>

## Revoke invitation

const result = yield* client.call("invitation.revoke", {

## Update invitation

Updates an existing invitation. Can be used to change invitation status, role, or other properties.

<ParamField body="id" type="string" required>
  Invitation ID to update.
</ParamField>

<ParamField body="status" type="string">
  New status: `"pending"`, `"accepted"`, `"expired"`, or `"revoked"`.
</ParamField>

<ParamField body="acceptedAt" type="string">
  ISO 8601 timestamp of acceptance.
</ParamField>

<ParamField body="acceptedBy" type="string">
  User ID who accepted the invitation.
</ParamField>

### Response

<ResponseField name="data" type="object" required>
  The updated invitation object.
</ResponseField>

<ResponseField name="transactionId" type="string" required>
  Transaction ID for optimistic UI updates.
</ResponseField>

### Errors

- `InvitationNotFoundError` - Invitation does not exist
- `UnauthorizedError` - User lacks permission
- `InternalServerError` - Unexpected server error

<CodeGroup>

```typescript TypeScript
const result = yield* client.call("invitation.update", {
  id: "inv_123",
  status: "accepted",
  acceptedAt: new Date().toISOString(),
  acceptedBy: "user_456"
});
```

```javascript JavaScript
const result = await client.call("invitation.update", {
  id: "inv_123",
  status: "accepted",
  acceptedAt: new Date().toISOString(),
  acceptedBy: "user_456"
});
```

</CodeGroup>
