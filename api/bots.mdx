## Overview

The Bots API allows you to create custom bots with specific permissions (scopes), manage bot lifecycle, regenerate API tokens, and provide a marketplace for installing public bots. Bots can respond to commands, send messages, and integrate with external services via webhooks.

## Bot Scopes

Scopes define what permissions a bot has:

- `messages:read` - Read messages in channels
- `messages:write` - Send messages to channels
- `channels:read` - Read channel information
- `channels:write` - Create and modify channels
- `users:read` - Read user information
- `reactions:write` - Add reactions to messages
- `commands:register` - Register slash commands

## Bot Management

These methods are for bot creators to manage their bots.

### bot.create

Create a new bot with specified scopes and permissions. Returns a plain API token that is only shown once.

<ParamField path="name" type="string" required>
  Bot name (1-100 characters)
</ParamField>

<ParamField path="description" type="string">
  Bot description (max 500 characters)
</ParamField>

<ParamField path="webhookUrl" type="string">
  URL to receive bot events and command invocations
</ParamField>

<ParamField path="scopes" type="BotScope[]" required>
  Array of permission scopes for the bot
</ParamField>

<ParamField path="isPublic" type="boolean" default={false}>
  Whether the bot is listed in the public marketplace
</ParamField>

<ResponseField name="data" type="Bot">
  The created bot object
  <Expandable title="properties">
    <ResponseField name="id" type="BotId">
      Unique bot identifier
    </ResponseField>
    <ResponseField name="userId" type="UserId">
      ID of the machine user created for this bot
    </ResponseField>
    <ResponseField name="createdBy" type="UserId">
      ID of the user who created the bot
    </ResponseField>
    <ResponseField name="name" type="string">
      Bot name
    </ResponseField>
    <ResponseField name="description" type="string | null">
      Bot description
    </ResponseField>
    <ResponseField name="webhookUrl" type="string | null">
      Webhook URL for events
    </ResponseField>
    <ResponseField name="scopes" type="string[] | null">
      Array of permission scopes
    </ResponseField>
    <ResponseField name="isPublic" type="boolean">
      Whether bot is public in marketplace
    </ResponseField>
    <ResponseField name="installCount" type="number">
      Number of organizations that have installed this bot
    </ResponseField>
    <ResponseField name="mentionable" type="boolean">
      Whether the bot can be @mentioned
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="token" type="string">
  Plain API token - only returned on creation, store securely
</ResponseField>

<ResponseField name="transactionId" type="TransactionId">
  Transaction ID for optimistic updates
</ResponseField>

**Errors:**
- `UnauthorizedError` - User is not authenticated
- `RateLimitExceededError` - Too many requests
- `InternalServerError` - Server error

<CodeGroup>
```typescript Example
const result = await client.rpc("bot.create", {
  name: "GitHub Bot",
  description: "Sends GitHub notifications to channels",
  webhookUrl: "https://example.com/bot/webhook",
  scopes: ["messages:write", "channels:read"],
  isPublic: true
})

console.log("Bot created:", result.data.id)
console.log("API Token (save this!):", result.token)
```
</CodeGroup>

## Bot Marketplace

These methods are for users to discover and install public bots.

### bot.listPublic

Browse all public bots available for installation. Returns bots with their installation status for your organization.

<ParamField path="search" type="string">
  Optional search query to filter bots
</ParamField>

<ResponseField name="data" type="PublicBotInfo[]">
  Array of public bots
  <Expandable title="PublicBotInfo properties">
    <ResponseField name="...Bot fields" type="Bot">
      All standard Bot fields
    </ResponseField>
    <ResponseField name="isInstalled" type="boolean">
      Whether your organization has installed this bot
    </ResponseField>
    <ResponseField name="creatorName" type="string">
      Name of the user/org that created the bot
    </ResponseField>
  </Expandable>
</ResponseField>

**Errors:**
- `UnauthorizedError` - User is not authenticated
- `InternalServerError` - Server error

<CodeGroup>
```typescript Example
const result = await client.rpc("bot.listPublic", {
  search: "github"
})

for (const bot of result.data) {
  console.log(`${bot.name} by ${bot.creatorName}`)
  console.log(`Installed: ${bot.isInstalled}`)
}
```
</CodeGroup>
